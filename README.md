## 🚀 Key Projects

<details>
<summary><b>🔷 Project 1 — Large-Scale On-Premises to AWS Cloud Migration</b></summary>

> **Stack:** AWS MGN · AWS DMS · EC2 · RDS (MSSQL) · S3 · EBS · VPC · Route 53 · ACM · AWS Backup · IIS

**Objective:** Migrate a legacy business-critical application and 1 TB MSSQL database from on-premises to AWS with minimal downtime.

| Phase | Action | Tool |
|---|---|---|
| Design | Multi-tier VPC, subnets, NACLs, SGs — HLD + LLD | Visio / Draw.io |
| Server Migration | Lift-and-shift of Windows & Linux servers | AWS MGN |
| DB Migration | MSSQL migration with continuous replication | AWS DMS |
| App Hosting | Windows app on EC2 + IIS + ACM SSL | EC2 / IIS / ACM |
| DNS | Zone migration and record management | Route 53 / GoDaddy |
| DR | Automated snapshots + cross-region replication | AWS Backup |

**Result:** ✅ 30% performance improvement · Zero data loss · 45-min cutover window · Cost-optimized foundation

📁 See: [`aws-cloudformation-templates/vpc/`](./aws-cloudformation-templates/vpc/) · [`cloud-migration-playbooks/aws-migration/`](./cloud-migration-playbooks/aws-migration/)

</details>

<details>
<summary><b>🔷 Project 2 — Replatform & Rearchitecture: Monolith to Cloud-Native</b></summary>

> **Stack:** Elastic Beanstalk · RDS Aurora (Multi-AZ) · Lambda · EventBridge · API Gateway · WAF · ACM · CloudWatch

**Objective:** Replatform a legacy Java app to AWS managed services and rearchitect batch workloads to serverless.

- ✅ App tier → **Elastic Beanstalk** (Auto Scaling + ALB); DB → **RDS Aurora Multi-AZ**
- ✅ Batch jobs → **Lambda + EventBridge** — 60% compute cost reduction
- ✅ **API Gateway + WAF + ACM SSL** for secure endpoints
- ✅ CloudWatch dashboards + alarms for full observability

**Result:** ✅ 99.9% availability SLA · Eliminated manual patching · Improved deployment velocity

</details>

<details>
<summary><b>🔷 Project 3 — Blue/Green Deployment with ALB & Auto Scaling</b></summary>

> **Stack:** ALB · Auto Scaling Groups · EC2 · CloudWatch · CloudFormation

**Objective:** Implement minimal-downtime deployment strategy for a high-traffic web application.

- ✅ ALB routes traffic between Blue (live) and Green (staging) environments
- ✅ Independent ASGs with CPU-based scaling per environment
- ✅ Instant rollback by rerouting ALB target group weights
- ✅ Full infrastructure defined in **CloudFormation**

**Result:** ✅ Zero deployment windows · Instant rollback · Improved release confidence

📁 See: [`aws-cloudformation-templates/alb/`](./aws-cloudformation-templates/alb/)

</details>

<details>
<summary><b>🔷 Project 4 — M365 Tenant-to-Tenant Email Migration (~500 Mailboxes)</b></summary>

**Objective:** Post-acquisition consolidation of two M365 tenants — mailboxes, calendars, contacts — with minimal disruption.

**Result:** ✅ 500 mailboxes migrated over a weekend · Complete data fidelity · Minimal user disruption
